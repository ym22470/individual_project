================================================================================
FINDING 4 — IS THE FINE-TUNING DECISION VALID?
================================================================================

Yes. The evidence strongly supports the decision to fine-tune the baseline
model toward longer, more detailed outputs:

1. Long model outperforms Base on every query type and every metric in both
   single-target and multi-target evaluations.
2. The gains are statistically proven (*** p<0.001 overall) on single-target
   across Recall@1, NDCG@5, and MRR. This is not noise.
3. The gains are largest where they matter most — Motion (*** p<0.001),
   Paraphrase (*** p<0.001), and Style (*** p<0.001 on NDCG@5) — the query
   types that differentiate a capable retrieval system from a basic matcher.
4. The Tags model (fine-tuned in the opposite direction) is statistically
   equivalent to Long only on Exact (ns). All other query types show
   significant degradation (** or *), proving concise descriptions are
   insufficient for non-literal queries.
5. The latency trade-off (+43 ms query latency, ~8.5s inference per video)
   is acceptable given the retrieval gains.

================================================================================
FINDING 5 — WHY STYLE QUERIES ARE HARD (AND HOW TO FIX IT)
================================================================================

Root cause: All three models are trained to describe WHAT is happening (objects,
actions, scene content). None reliably describe HOW it looks (lighting, mood,
colour grading, cinematographic style). Without style vocabulary in the
descriptions, neither semantic search nor keyword matching can bridge the gap.

Hybrid search (dense + BM25 keyword) is unlikely to resolve this — if no
description mentions "golden hour" or "high contrast", BM25 has nothing to
match on. The fix is at the description layer, not the retrieval layer.

Recommended next steps:
- Add a style-focused secondary prompt: "Describe the visual style, lighting,
  colour palette, and mood of this video."
- Evaluate whether a two-pass indexing strategy (content description + style
  description stored separately) improves style query retrieval.
- Consider fine-tuning on cinematography-annotated data to teach the model
  to surface atmospheric and stylistic attributes.

================================================================================
FINDING 6 — HYBRID SEARCH (BM25 + HNSW COSINE VIA RRF)
================================================================================

Hybrid search was evaluated by combining BM25 keyword ranking and ChromaDB
HNSW cosine ranking using Reciprocal Rank Fusion (RRF, k=60).

Average metric delta over pure cosine baseline:

  Model   Recall@1 Δ   MRR Δ    Multi-target MAP Δ
  ---------------------------------------------------
  Base      -0.022     -0.036         -0.056
  Long      +0.049     +0.033         -0.022
  Tags      -0.014     -0.017         -0.022

Key observations:

1. Only the Long model benefits from hybrid search on single-target queries.
   BM25 requires vocabulary overlap between query and document. Long's verbose
   descriptions are rich enough to contain query keywords; Base and Tags
   descriptions are too short or too structured, so BM25 adds noise.

2. Style queries are the one consistent winner across all models in both
   benchmarks. The Long model's single-target Style Recall@1 improves from
   0.405 → 0.514 and MRR from 0.561 → 0.623. Style queries occasionally
   contain distinctive atmospheric words that embed poorly but match well
   as keywords, making this the most promising use case for hybrid search.

3. Paraphrase queries degrade significantly with hybrid search — proven by
   significance testing (** p<0.01 on NDCG@5 and MAP in multi-target).
   Paraphrase queries deliberately use different vocabulary from descriptions,
   so BM25 penalises the correct documents. This is not random variation.

4. Hybrid improvements on Exact are statistically significant (* p<0.05 on
   NDCG@5, Recall@1, and MRR in single-target). Style Recall@1 improvement
   is also significant (* p=0.011). These are the only query types where
   hybrid search is justified.

5. Multi-target overall MAP shows no significant hybrid improvement (ns),
   driven by Paraphrase degradation outweighing Style gains.

Recommendation: Hybrid search is not universally beneficial. A query-type-
aware routing strategy is supported by significance testing:
  - Use hybrid for Exact (* proven improvement) and Style (* on Recall@1)
  - Use pure cosine for Paraphrase (** proven degradation) and Semantic (ns)

  Base 0.635 / Long 0.688 / Tags 0.644

Key observations:

  metric shows Long not only retrieves correct videos more often, but ranks
  them higher within the result list.
- Multi-target Style NDCG@5 is flat across models (0.336-0.339), independently
  confirming the description quality ceiling finding.
- NDCG@1 on multi-target Style shows Tags (0.350) edges Long (0.250),
  revealing a nuance invisible to MAP and Recall: Tags occasionally places
  a relevant style video at rank 1 more reliably, while Long distributes
  relevant results deeper in the list. By NDCG@10 Long recovers (0.408 vs
  0.386), suggesting Long has broader coverage at the cost of top-1 precision
  on style queries.

================================================================================
OVERALL CONCLUSION
================================================================================

The long_quantized model is the best-performing model across all query types,
evaluation settings, and metrics. Its superiority over Base is statistically
proven (*** p<0.001 overall on single-target; * p<0.05 overall on multi-target)
and is not a sample size artefact.

The fine-tuning direction — longer, richer video descriptions — is empirically
validated and statistically confirmed. The evidence is consistent across
Recall@1, Recall@5, MRR, NDCG@K, and MAP.

The primary remaining challenge is style-based retrieval. All models plateau
at ~0.4 NDCG@5 on style queries regardless of description length or search
algorithm. Significance testing confirms this is a description generation
problem: the models do not describe visual style or cinematographic attributes.
Hybrid search offers a statistically significant but partial improvement on
style Recall@1 (* p=0.011) but does not close the gap.

Priority next steps ranked by expected impact:
  1. Style-focused description prompt (highest impact, lowest effort)
  2. Query-type-aware routing: hybrid for Exact/Style, cosine for Paraphrase
  3. Two-pass indexing: separate content and style descriptions per video
  4. Fine-tuning on cinematography-annotated data (highest effort, highest
     ceiling)
